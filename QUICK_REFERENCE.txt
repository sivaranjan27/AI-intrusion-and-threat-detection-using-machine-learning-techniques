TAB 1: LOGIN & ATTACK DETECTION
────────────────────────────────
Purpose:    Test login attempts and detect attack patterns
Default:    Username: admin, Password: admin123

How it works:
1. Upload Excel file with username/password columns (optional)
2. Enter username and password
3. Click Login
4. System tracks failed attempts:
   - 1-5 failed attempts: Warning
   - 6+ failed attempts: ATTACK DETECTED
5. Attack type determined:
   - BRUTE-FORCE: Generic multiple failed attempts
   - DICTIONARY: Failed attempts using common passwords
   - RAINBOW: 8+ failed attempts (precomputed hashes)
6. Email alert sent to admin

Test it:
→ Try logging in 6 times with wrong password to see attack detection
→ Use common password like "password123" to trigger dictionary attack
→ Use 8+ attempts to trigger rainbow attack

TAB 2: PHISHING DETECTION  
────────────────────────────
Purpose:    Analyze URLs for phishing characteristics
Input:      Any URL starting with http:// or https://

Risk Analysis (6 Features):
┌─ URL Length          (15%): >75 chars = suspicious
├─ HTTPS               (25%): Missing = high risk
├─ Domain              (20%): Unknown/IP address = suspicious
├─ Special Chars       (20%): @ # $ % in domain = risk
├─ Keywords            (15%): phishing words in URL
└─ Subdomains          (5%): >2 subdomains = anomaly

Risk Score Interpretation:
  0-30%:   Likely Legitimate (Safe)
  30-70%:  Suspicious (Review carefully)
  70-100%: Likely Phishing (Avoid!)

Test URLs:
✓ Safe:        https://www.google.com
⚠ Suspicious:  https://secure-verify-account.example.com
✗ Phishing:    http://g00gle-security.verify.com

TAB 3: DOS INFORMATION
──────────────────────
Purpose:    Educational content about DOS/DDoS attacks

5 Attack Types:
  1. Volumetric Attacks    - Flood with massive traffic
  2. Protocol Attacks      - Exploit weaknesses
  3. Application Layer     - Target web apps
  4. Botnet Attacks        - Use infected computers
  5. DNS Amplification     - Amplify attack traffic

5 Prevention Strategies:
  1. Rate Limiting         - Limit requests per IP
  2. Traffic Filtering     - Block malicious traffic
  3. Infrastructure        - Use CDN, distributed servers
  4. Detection             - Monitor and alert
  5. Response Procedures   - Incident response plan

Key Contacts:
  FBI Cyber:        Local field office
  ISP Support:      Your internet provider
  CERT/CC:          cert@cert.org
  DDoS Services:    Cloudflare, AWS Shield, Akamai
